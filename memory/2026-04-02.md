# 2026-04-02 ‚Äî dev-nexus WIF Deployment Fix (Evening Session)

## What We Did

### Root Cause Found: WIF Provider Missing `attribute.repository` Mapping

The GitHub Actions workflow was failing at `google-github-actions/setup-gcloud` with `Permission 'iam.serviceAccounts.getAccessToken' denied`. After extensive debugging:

**Root cause:** The WIF provider's `attribute_mapping` did NOT include `attribute.repository`. The `serviceAccountTokenCreator` IAM binding used `principalSet://.../attribute.repository/patelmm79/dev-nexus` ‚Äî but since `attribute.repository` was never mapped from the OIDC token, the binding could never match.

**Two fixes applied:**
1. Added `attribute.repository = "assertion.repository"` to WIF provider's attribute_mapping in Terraform (`terraform/github_actions_wif.tf`)
2. Terraform apply in Cloud Shell applied the fix to GCP

**The attribute_condition passed** (`assertion.repository == "patelmm79/dev-nexus"`) ‚Äî but that's evaluated at WIF admission time, not at token minting time. The binding evaluation uses the attribute_mapping, not the condition directly.

### WIF Architecture Now Working

- GitHub Actions ‚Üí WIF pool ‚Üí `workloadIdentityUser` binding ‚Üí impersonates SA ‚úÖ
- GitHub Actions ‚Üí WIF pool ‚Üí `serviceAccountTokenCreator` binding (attribute-based) ‚Üí can mint tokens ‚úÖ
- OIDC token correctly contains `repository: "patelmm79/dev-nexus"` ‚úÖ

### Terraform Apply in GitHub Actions ‚Äî Secrets Architecture

Resolved how to pass Terraform variables without committing `tfvars` files to the repo.

**Pattern established:**
- **Secrets** (sensitive values): GitHub ‚Üí Settings ‚Üí Environments ‚Üí `{env}` ‚Üí Secrets
  - `PROD_GITHUB_TOKEN`, `PROD_ANTHROPIC_API_KEY`, `PROD_POSTGRES_PASSWORD`, `PROD_GITHUB_CLIENT_ID`, `PROD_GITHUB_CLIENT_SECRET`, `PROD_JWT_SECRET`, `PROD_FRONTEND_URL`
- **Environment variables** (non-secret config): Same GitHub environment ‚Üí Variables section
  - `WIF_PROVIDER`, `WIF_SERVICE_ACCOUNT`, `PROD_FRONTEND_URL`
- **Workflow YAML** (env-specific non-secret config): hardcoded in workflow `env:` block
  - CPU, memory, instance counts, region, allowed origins, etc.

### Workflow Files Created

- `.github/workflows/terraform-plan.yml` ‚Äî plan on PR + manual `workflow_dispatch` trigger
- `.github/workflows/terraform-apply.yml` ‚Äî apply on push to main + manual `workflow_dispatch`

Both use WIF auth, `hashicorp/setup-terraform@v3`, and pull secrets from GitHub Actions `prod` environment.
