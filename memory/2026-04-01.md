## What is GlobalBitings
A dish-first restaurant discovery app. Core idea: user reads about/tries a dish → app finds where to eat it in their city.

**3-phase vision:**
1. MVP: dish search by city → restaurant attribution
2. Discovery: recommendation engine based on travel history + taste profile
3. Expert layer: food critics/bloggers contribute curated dish guides

**Unique value prop:** Google/Yelp are restaurant-first. GlobalBitings is dish-first — "I loved Schiacciata in Siena, where do I get THAT in NYC?" No platform solves this well today.

## Performance recommendations (adopted after 12:49 UTC session)
1. **Memory as working log** — write to `memory/YYYY-MM-DD.md` at the START of any non-trivial task, not just at end/compactions
2. **Active file tracking** — when editing multiple files in a session, note which ones are "in progress" so a reset doesn't lose track
3. **Check session transcript on "status?"** — if memory is empty but previous sessions exist, always check `sessions/` to reconstruct before answering "I don't know what I was working on"
4. **Git commit early and often** — commit partial work, not just complete work. A half-built script that's committed is recoverable; one only in memory is lost
5. **Batch similar operations** — multiple file writes, git operations, or web searches in one turn are cheaper than spreading them across sessions
