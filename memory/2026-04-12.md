## Summary Expander "Narrow" Fix (02:11 UTC)

### Problem
Analysis page: Elastica Shape & Angle expander plot appears "narrow" — x-axis window ~44" for a ~27" span curve.

### Root cause
`plot_elastica_shape()` used `arclength` for x-axis range calculation. The actual rod arc covers `arclength` but the visible curve only spans `forced_span` (press displacement). `forced_span` was passed to the function but never used for x-axis range.

### Fix
In `src/plotting_plotly.py`:
- `plot_elastica_shape()`: changed `xlim = (-0.15 * arclength + 0.125, 1.15 * arclength + 0.125)` to `xlim = (-0.15 * span + 0.125, 1.15 * span + 0.125)` using `forced_span` (local var `span`)
- Both `plot_elastica_shape()` and `plot_summary()` now use identical x-axis range logic based on `forced_span`

## resume-customizer LLM Provider Routing Fix (02:30–03:00 UTC)

### Root Bug
User's selected LLM provider/model (Claude, OpenRouter, etc.) was **silently ignored** by the V2 pipeline, causing fallback to defaults and `models/claude-haiku-4-5 is not found` errors.

### Root Cause — threading.local() is per-thread, NOT inherited by ThreadPoolExecutor workers
The code used `set_agent_llm_context(provider, model)` → `threading.local()` → `get_agent_llm_client()` inside agents. But `ThreadPoolExecutor` workers are **new threads** that don't inherit the main thread's `_thread_local` — each worker thread gets a fresh empty `_thread_local`. So agent `__init__` inside worker threads saw no provider/model → fell back to defaults.

### Fix — Pass provider/model through WorkflowStateV2 instead of thread-locals
LangGraph state dict IS passed into the worker thread (it flows through the graph), so `llm_provider`/`llm_model` fields in state are the reliable vehicle.

#### Files Modified
1. **`workflow/state_v2.py`**: Added `llm_provider: Optional[str]` and `llm_model: Optional[str]` fields to `WorkflowStateV2`. Updated `create_initial_state_v2()` to accept/populate these.

2. **`workflow/orchestrator_v2.py`**: `V2WorkflowOrchestrator.start_analysis()` now accepts `llm_provider`/`llm_model` and passes them to `create_initial_state_v2()`. Also uses `inspect` for backwards-compat with callers that don't pass these args.

3. **`utils/agent_helper.py`**: `get_agent_llm_client()` — new `workflow_state` parameter as first fallback (before thread-local, session state, defaults). Precedence: (1) workflow_state, (2) thread-local, (3) session state, (4) defaults.

4. **`workflow/nodes_v2.py`**: All node functions (`v2_job_analysis_node`, `v2_priority_ranking_node`, `v2_alignment_builder_node`, `v2_top_skills_extractor_node`, `v2_gap_analyzer_node`, `v2_title_adapter_node`, `v2_summary_adapter_node`, `v2_skills_adapter_node`, `v2_highlights_adapter_node`, `v2_work_history_adapter_node`, `v2_earlier_career_adapter_node`, `v2_top_third_optimizer_node`, `v2_scorer_node`, `v2_validator_node`, `v2_freeform_editor_node`) — all now pass `workflow_state=state` to agent constructors.

5. **All 15 V2 agent `__init__` methods** — all now accept `workflow_state: dict = None` and pass it to `get_agent_llm_client(workflow_state=workflow_state)`. Agents: `V2JobAnalyzer`, `V2PriorityRanker`, `V2AlignmentBuilder`, `V2TopSkillsExtractor`, `V2GapAnalyzer`, `V2TitleAdapter`, `V2SummaryAdapter`, `V2SkillsAdapter`, `V2HighlightsAdapter`, `V2WorkHistoryAdapter`, `V2EarlierCareerAdapter`, `V2TopThirdOptimizer`, `V2Scorer`, `V2Validator`, `V2FreeformEditor`, `V2UseCaseSelector`.

6. **`main.py`**: `ResumeCustomizer.start_workflow()` now passes `provider`/`model` directly to `orchestrator.start_analysis()`. Removed now-useless `set_agent_llm_context`/`clear_agent_llm_context` wrapper (though the functions themselves remain as no-ops).

7. **`tests/test_llm_provider_routing.py`**: Created comprehensive regression tests (3 tests: provider+model flow, fallback chain, V2 nodes workflow_state passthrough).
