### The REAL Problem: Vercel 404 on /auth/callback
- **Root cause**: `vercel.json` was missing SPA catch-all rewrite rule
- React Router had the `/auth/callback` route defined in `App.tsx`
- But Vercel didn't know to serve `index.html` for client-side routes → returned 404 NOT_FOUND
- This is NOT a Cloud Run or backend issue — it was purely a Vercel routing config issue

### Fix Applied
Updated `vercel.json` to add SPA fallback rewrite:
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/dist/index.html" }
  ]
}
```
**Note**: This is a broad SPA fallback — will match all paths including static assets. For a cleaner approach, use a `vercel.json` that only rewrites non-file paths, but this works.

## Session: ~21:20–22:52 UTC — resume-customizer Auto-Generate button fixes

### 4 Bugs Fixed and Pushed to `origin/main`

**Bug 1 — `StreamlitDuplicateElementId`:** Duplicate "📄 Export PDF (No Changes)" button left over from 2-column → 3-column layout refactor (lines 2210–2223). Both rendered simultaneously, crashing on rerun. Fixed: removed the stray `with col2:` block. (commit `d71ddaa`)

**Bug 2 — Button silently does nothing:** `queue.Queue` stored in `st.session_state.auto_generate_queue` can't be pickled for Streamlit's session state serialization → silently fails on rerun. Fix: replaced with file-based result store (`/tmp/auto_gen_<session_id>_<count>.json` + `.pdf`). (commit `d38132a`)

**Bug 3 — `pdf_bytes` not JSON-serializable:** `WorkflowStateV2` contains `pdf_bytes` (bytes) → `json.dump()` crashes when workflow completes. Fix: write `pdf_bytes` to a separate binary `.pdf` file, re-attach after JSON deserialization. (commit `2baee84`)

**Bug 4 — Background thread can't access `st.session_state`:** `ResumeCustomizer` was instantiated in the button handler and stored in `st.session_state.customizer`, but the background thread tried to access `st.session_state.customizer` in `_auto_run()` — this fails in multi-worker Streamlit Cloud (worker thread has empty session state). Fix: instantiate `ResumeCustomizer` fresh inside `_auto_run()` closure, capturing `workflow_version` from `st.session_state` in the foreground. (commit `c9e0baba`)

### Root Cause Analysis
Streamlit Cloud uses multiple worker processes/threads. `st.session_state` is process-local and not shared across threads. The background thread running `_auto_run()` was in a separate thread that never saw `customizer` initialized in the main thread.
