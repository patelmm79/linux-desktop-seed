## GitHub Login OAuth Fixes (14:36–16:47 UTC)

### Problem: GitHub OAuth CSRF mismatch / redirect_to issues

**Issue 1 — Wrong API URL in frontend:** `.env.production` pointed to wrong Cloud Run URL. Fixed to `pattern-discovery-agent-75l7mntama-uc.a.run.app`. Also Vercel project settings had `VITE_API_BASE_URL` set to `http://localhost:3000` in GUI — overrode the file. User had to fix this manually in Vercel dashboard.

**Issue 2 — CORS origin wrong on backend:** `allowed_origin_regex` was placeholder `your-production-frontend\.vercel\.app`. Fixed to `dev-nexus-frontend\.vercel\.app`. Commit `3e8e5d1`.

**Issue 3 — CSRF state cookie stripped by Cloud Run:** Google Frontend proxy was stripping cookies (even with SameSite=None). Solution: pass CSRF state via URL instead of cookie. GitHub passes state back unchanged through the OAuth redirect.

**Issue 4 — redirect_uri had literal `{code}` placeholder:** The callback URL had `?code={code}` which broke GitHub's append of `?code=...&state=...`. Fixed to use clean callback URL that GitHub appends to.

**Issue 5 — FRONTEND_URL not set in container:** After OAuth, backend redirected to `http://localhost:3000` instead of Vercel frontend. Added `FRONTEND_URL=https://dev-nexus-frontend.vercel.app` to deploy.

### OAuth Flow Summary (how it works now)
1. Frontend calls `GET /auth/github` (no auth needed)
2. Backend redirects to GitHub with `redirect_uri=https://pattern-discovery-agent-75l7mntama-uc.a.run.app/auth/github/callback?redirect_to=<encoded_frontend_url>`
3. GitHub redirects user back to callback with `?code=...&state=...` appended
4. Backend exchanges code for token, creates JWT, redirects to `https://dev-nexus-frontend.vercel.app/auth/callback?token=<jwt>`
5. Frontend stores JWT and considers user logged in
