## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, use one of these methods:

1. **GitHub Security Advisories** (recommended)
   - Go to the [Security tab](https://github.com/patelmm79/linux-desktop-seed/security/advisories/new)
   - Click "Report a vulnerability"
   - Fill out the vulnerability report form

2. **Email**
   - Email the maintainer directly (if listed in the repository)

### What to Include

Please include as much of the following as possible:
- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact assessment

## Response Timeline

- **Acknowledgment**: Within 7 days
- **Initial Assessment**: Within 14 days
- **Fix Timeline**: Depends on severity — critical issues get priority

## Disclosure Policy

- We follow a **coordinated disclosure** process
- Public disclosure will only be made after a fix is available
- We credit reporters in the security advisory (with permission)

## Security Best Practices

When using this project, remember:

- **Never hardcode credentials** in deployment scripts
- Use environment variables for sensitive data
- Review the `SECURITY.md` in the repository for any deployment-specific concerns
- Keep your deployment environment updated

## Thanks

Thank you for helping keep this project and its users safe!
